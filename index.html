CU Short URLs

A new service provided by SRC Web Team

Hi everyone, thanks for giving your time to listen to this presentation. My name is Brooke and I'm a developer on the team. I'm here to propose a new service, CU Short URLS, which we think will be useful for students, staff, and content creators.

This service allows users to create short urls that match the university brand. We've created a new D9 site to interface with YOURLs, an open source link shortener.

We can go from a url like https://www.colorado.edu/academics/programs/technology-arts-media ,
to something like https://sho.rt/tam

We've all seen long, complex urls before. They're hard to remember and not particularly great to look at. Link shorteners can turn those long urls into a much shorter one. The short links are easy to remember and look better when added to a presentation, post, or site. This service will help unify content under the CU brand because the links will retain a CU domain instead of using an external one like bit.ly. [next]

For example, we can turn a long url like this one, which is a link to the Technology, Arts, and Media program page into something like /tam.

Feature Overview

  • New Roles
  • Generate random URLs like https://sho.rt/f56j from any approved domain
  • Applications for shortening non approved domains or creating custom paths
  • Block to show a list of existing short links
  • Automatic update emails sent to applicants

The link shortener includes some features that will make managing it easier. We've added new roles and permissions, which I'll explain on the next slide.

Next, this service can be used two ways. The first is to generate random short links like the ones created with bitly, however, those urls will be restricted to a list of allowed domains only.

The second way is to create custom short links, which have either custom pathnames or shorten urls from non approved domains. Generating these links require users to fill out a short webform. The applications provide a way for admins to verify that the link is appropriate and to prevent collisions between keywords. Emails are also sent automatically to applicants about their application status.

Last, there's a block that shows a list of existing short links.

Roles

New users will receive the Web Service Contributor role. This allows them to generate random URLs and apply for custom ones

The Web Service Admin role allows them to view stats about existing short URLs and manage applications

If users are abusing the service, admins can remove the Web Service Contributor role, restricting their access

We created two new roles for this site. When a user authenticates for the first time, they will receive the Web Service Contributor Role. They can generate random urls and apply for custom ones. [next]

Web Service Admins can approve, reject, and delete applications. They will also have access to the YOURLs installation where they can view more stats about the short links. [next]

Furthermore, if a user is abusing the service, an admin can remove the Web Service Contributor role, which will revoke their access.

Configuration

Web Service Admins can easily connect Drupal to YOURLs

The site uses the YOURLs API, so there's a little bit of setup to get the service fully running.

The configuration form needs the API endpoint and the key. There's also a series of inputs that list the allowed domains for creating random short links. Last, there's a checkbox which enables sending emails. Normally, it should be checked, but it can be useful to turn it off for testing.

Let's generate a random URL

Creating a random URL is very simple. Users will copy and paste a url into the form and a short link will be created! Remember, these URLs must come from an approved domain. Also, if a user tries to input a url that's already been shortened, the existing short link will be returned. This is because there can't be any duplicate long or short urls.

Now let's Apply for a Custom URL

Users will fill out this form with a few details about the URL they want to shorten

Ex) A user could ask to shorten:

https://cubuffs.com/sports/2017/7/28/ticket-information-request-form.aspx

to

/tickets

The application for creating custom urls is pretty short. Users will have to give the site name, long URL, keyword, and a reason for applying. For example, one could ask to shorten this long url, which links to the page for buying sporting event tickets, to just /tickets. After they submit the form, they'll get a confirmation message and will have to wait for their request to be approved.

Managing URL Applications

Once a user has requested a short URL, an admin will either Approve or Reject it. Emails are automatically sent to applicants notifying them of their status.

Once a user has submitted an application, an admin will either approve or reject it.

To approve an application, admins will edit the webform submission. At the bottom, they can change the Application Status to either Approved or Rejected. Applications that still need a decision will have the status Pending. Upon approval, a new short link will be created. If an admin deletes a submission, the short link associated with it will also be deleted (if it's been created). This also means that the keyword is free to use again.
